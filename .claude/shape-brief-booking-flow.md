# Design Brief: 场馆预订流程

## 1. Feature Summary

体育场馆预订流程，3步完成：选日期时段 → 确认信息 → 预订成功。面向从容规划型用户，提供丰富的场馆信息（照片、设施、取消政策、评价），帮助用户做出自信的预订决策。移动端优先，高保真实现。

## 2. Primary User Action

用户在浏览场馆详情后，选择合适的日期和时段，确认预订信息，完成预订。核心目标：让用户从"这个场馆不错"到"我订好了"的过程清晰、流畅、有掌控感。

## 3. Design Direction

- **Color Strategy**: Restrained（沿用 DESIGN.md 定义的 tinted neutrals + teal/blue accent）
- **Theme Scene**: 用户在周末下午坐在沙发上，用手机规划下周的运动安排，屏幕亮度中等，心态从容但希望操作高效。
- **Anchor References**:
  - ClassPass（预订流程的简洁步骤感）
  - Airbnb（丰富的场馆信息展示 + 清晰的价格/政策）
  - Apple Bookings（干净的时间选择交互）

## 4. Scope

- **Fidelity**: High-fi（高保真，覆盖主要状态，精致细节）
- **Breadth**: 一个完整流程（3个步骤/页面）
- **Interactivity**: 可交互原型，包含状态切换、动画过渡
- **Time intent**: 按生产级质量打磨，但先聚焦成功路径

## 5. Layout Strategy

**垂直流布局，信息密度适中。**

- **Step 1（选时间）**: 顶部场馆摘要（照片+名称+地址），下方日期横向滚动选择，再下方时段网格。视觉重心在时间选择上。
- **Step 2（确认）**: 预订摘要卡片（场馆、时间、价格），下方展示取消政策和设施亮点。底部固定"确认预订"按钮。
- **Step 3（成功）**: 成功确认动画 + 预订详情卡片 + 两个CTA（查看我的预订 / 继续浏览场馆）。

移动端单列布局，桌面端可考虑左侧信息+右侧操作的双栏。

## 6. Key States

| 状态 | 说明 |
|------|------|
| **Default** | 各步骤的正常展示状态 |
| **Loading** | 时段加载中（骨架屏）、提交预订中（按钮loading态） |
| **Empty** | 所选日期无可用时段（友好提示+推荐其他日期） |
| **Error** | 网络错误、预订失败（清晰的错误信息+重试按钮） |
| **Success** | 预订完成确认页 |
| **Disabled** | 已过去的日期/时段不可选，视觉降级 |

## 7. Interaction Model

- **日期选择**: 横向滚动日期条，点击选中，已选日期高亮（accent色）
- **时段选择**: 网格布局，可选/已满/已选三种状态。点击可选时段立即选中，再次点击取消
- **步骤切换**: 步骤指示器在顶部，点击可返回上一步。步骤间使用滑动过渡动画
- **确认按钮**: 固定在底部，始终可见。点击后进入loading态，成功后过渡到完成页
- **完成页**: 微妙的成功动画（对勾绘制），预订详情卡片可展开

## 8. Content Requirements

- **场馆摘要**: 场馆名称、地址、运动类型、一张主图
- **日期选择**: 未来14天，显示星期和日期
- **时段**: 每小时一个时段（如 09:00-10:00），显示价格
- **取消政策**: 简短文案（如"预订后24小时内可免费取消"）
- **设施亮点**: 3-4个图标标签（如停车场、淋浴、器材租赁）
- **确认页**: 场馆名、日期、时段、时长、总价、取消政策
- **成功页**: 预订编号、场馆信息、时间、"已发送确认邮件"提示
