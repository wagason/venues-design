# 场馆详情页 — 设计 Brief

## 1. Feature Summary

场馆详情页是用户从搜索结果或分享链接进入后的决策页面。目标：5 秒内判断"这个场地值不值得去"。去掉时段预览和评价，聚焦核心决策信息：场馆是什么、在哪里、怎么联系、怎么到、有什么设施、多少钱。

## 2. Primary User Action

快速扫描 → 确认信息 → 点击预订。用户带着"差不多就订了"的心态进来，页面要做的就是扫除最后的犹豫。

## 3. Design Direction

| 项 | 选择 |
|---|---|
| Color strategy | Restrained（着色中性色 + 强调色承载 CTA 和关键数据） |
| Theme | 户外强光下快速读取信息，高对比度、大触摸目标 |
| Anchor references | 高德 POI 详情页（信息密度）、大众点评商户页（设施标签）、Apple Maps 地点卡（视觉克制） |

## 4. Scope

- **Fidelity**: Production-ready
- **Breadth**: 单页面
- **Interactivity**: 轮播滑动 + 地图交互 + 文字折叠
- **Time intent**: 上线质量

## 5. Layout（从上到下）

1. **照片轮播** — 4:3 比例，浮层返回/分享/收藏按钮，底部圆点指示器
2. **场馆身份区** — 场馆名称 + 地址 + 距离 + 运动类型徽章
3. **联系方式** — 手机号（可点击拨打）+ 微信号（可长按复制），图标引导
4. **地图卡** — 嵌入式静态地图 + "导航过去"按钮 + 距离/交通方式提示
5. **设施标签** — 横向滚动标签行（图标+文字），边缘渐隐，替代 4 列网格
6. **场馆介绍** — 超过 3 行折叠，点击"展开"查看完整
7. **底部固定 CTA** — 左侧价格（¥120/小时），右侧双按钮："预约录像"（次要样式）+ "预订场地"（主要样式）

## 6. Key States

| 状态 | 表现 |
|---|---|
| Default | 完整信息展示 |
| 轮播滑动 | 触摸滑动 + 指示器同步 |
| 长场馆名 | 允许换行，不截断 |
| 户外强光 | 高对比度，≥44px 触摸目标 |
| 介绍折叠/展开 | 平滑高度动画，ease-out-expo |

## 7. Interaction Model

- **轮播**：触摸滑动，底部指示器，首张 eager loading
- **手机号**：点击触发 `tel:` 链接
- **微信**：点击复制到剪贴板，toast 提示"已复制微信号"
- **地图**：点击跳转系统地图 app 导航
- **设施标签**：横向滚动，边缘渐隐提示
- **介绍文字**：超过 3 行折叠，点击展开/收起
- **底部 CTA**：固定视口底部，"预订场地"为主按钮，"预约录像"为次按钮
