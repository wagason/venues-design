# Design Brief: 预订流程（含视频服务）

> Shape 命令输出 — 2026-05-08
> 最后更新 — 2026-05-08
> 目标：booking-flow 完整流程，含视频服务选择

## 1. Feature Summary

运动场地预约平台的核心预订流程。用户选择运动类型和日期，在时间表网格中选择场地时段，为支持摄像头的场地选择视频服务（直播/录像），确认预约信息后完成预订。流程共 4 步，不支持视频的场地跳过第 2 步。

**流程总览：**

```
Step 1: 选择场地与时间
  ↓（场地支持视频）
Step 2: 选择视频服务 ← 新增
  ↓
Step 3: 确认预约
  ↓
Step 4: 预约成功

Step 1: 选择场地与时间
  ↓（场地不支持视频）
Step 3: 确认预约（跳过 Step 2）
```

## 2. Primary User Action

1. **Step 1**：选择运动类型 → 选择日期 → 在时间表网格中比较多个场地的可用时段，选择连续时段范围
2. **Step 2**：判断是否需要视频服务，在直播与录像之间选择（或跳过）
3. **Step 3**：核对预约信息（场地、时段、视频服务、价格），确认提交
4. **Step 4**：查看预约成功确认和收据，执行后续操作（添加日历、分享、继续浏览）

## 3. Design Direction

- **Color strategy**: Restrained — 着色中性色 + 强调色。信息密集型界面，颜色保持克制，用强调色高亮选中状态和关键 CTA。
- **Theme scene sentence**: 运动爱好者在户外阳光下，单手握手机，快速扫视多个场地的可用情况并完成预订，需要在明亮环境中清晰可读。
- **Anchor references**:
  - Google Calendar 周视图（时间轴 + 多列布局）
  - Apple 产品配置页（清晰的二选一对比卡片）
  - Airbnb add-on 选择（附加服务的叠加模式）
- **Per-surface override**: 无，沿用 DESIGN.md 中的 Restrained 策略。

- 运动类型选择器在最顶部，横向滚动 chips，默认选中足球

- 列头固定在顶部（sticky），滚动时保持可见
- 每个格子显示：价格 + 时间段（如 `¥200 09:00~10:00`）
- 已满格子显示"已满"
- 高峰时段：格子顶部2px暖色线条标识
- 选中状态：格子填充强调色，显示白色文字
- 支持范围选择：点击起始格子，再点击结束格子，中间格子高亮

## 6. Key States

### Step 1 状态

| 状态 | 用户需要看到什么 |
|---|---|
| 默认 | 显示足球运动的所有场地时间表，无选中时段 |
| 切换运动类型 | 网格更新为该运动的场地列表，重置已选时段和视频服务 |
| 切换日期 | 网格更新为该日期的可用情况，重置已选时段和视频服务 |
| 选中单个格子 | 格子高亮，底部显示价格汇总 |
| 范围选择 | 起始到结束的格子都高亮，显示总时长和总价 |
| 已满格子 | 灰色背景，"已满"文字，不可点击 |
| 高峰时段 | 格子顶部2px暖色线条，价格文字用高峰色 |
| 范围含已满时段 | Toast 提示"所选范围内包含已满时段，请重新选择" |

### Step 2 状态（视频服务）

| 状态 | 用户需要看到什么 |
|---|---|
| 默认（未选择） | 两张卡片均为未选中态，底部按钮显示场地原价 |
| 选中直播 | 直播卡片高亮（强调色边框 + 勾选），价格叠加到底部 |
| 选中录像 | 录像卡片高亮，价格叠加到底部 |
| 切换选择 | 点击另一张卡片自动切换（单选），价格实时更新 |
| 取消选择 | 再次点击已选中的卡片，回到未选状态 |
| 跳过 | 点击"不需要视频服务"，直接进入确认页 |

### Step 3 状态（确认）

| 状态 | 用户需要看到什么 |
|---|---|
| 有视频服务 | 预订详情中显示视频服务行（类型 + 单价 × 时长），总价包含视频费 |
| 无视频服务 | 预订详情不显示视频服务行，总价仅含场地费 |
| 提交中 | 确认按钮显示 loading 动画，禁用点击 |

### Step 4 状态（成功）

| 状态 | 用户需要看到什么 |
|---|---|
| 有视频服务 | 收据中显示视频服务行和费用 |
| 无视频服务 | 收据不显示视频服务行 |

## 7. Interaction Model

### Step 1 交互

- **运动类型切换**：点击顶部 chips 切换运动类型，网格即时更新，重置已选时段和视频服务
- **日期切换**：点击日期 chips 切换日期，网格即时更新，重置已选时段和视频服务
- **滚动**：网格区域400px固定高度，内容超出时垂直滚动，列头 sticky 保持可见
- **点击格子**：选中单个时段
- **再次点击**：取消选中（单个）或选择范围结束点
- **范围选择**：点击起始 → 点击结束 → 自动填充中间格子 → 检查范围内是否有已满时段
- **下一步**：场地支持视频 → 进入 Step 2；不支持视频 → 跳至 Step 3

### Step 2 交互（视频服务）

- **点击卡片**：选中该服务，另一张自动取消选中（单选）
- **再次点击已选中卡片**：取消选择，回到未选状态
- **价格实时更新**：底部栏即时反映场地费 + 视频费
- **跳过**：点击"不需要视频服务"，直接进入 Step 3
- **下一步**：进入 Step 3（无论是否选择视频服务）

### Step 3 交互

- **确认预约**：点击后按钮进入 loading 态，1.5秒后进入 Step 4
- **返回**：回到 Step 2（如有视频支持）或 Step 1（如无视频支持）

### Step 4 交互

- **添加日历**：Toast 提示"日历功能开发中"
- **分享预约**：调用 Web Share API 或 Toast 提示
- **查看我的预约**：Toast 提示"我的预约页面开发中"
- **继续浏览场馆**：跳转至 index.html

### 全局交互

- **返回按钮**：Step 1 和 Step 4 隐藏，Step 2/3 可见
- **键盘**：Escape 键返回上一步
- **步骤指示器**：顶部 4 个进度条，当前步高亮，已完成步半透明

## 10. Resolved Questions

- ~~移动端屏幕较小，4个场地列是否需要固定列宽？~~ → **固定列宽**，水平滚动浏览
- ~~范围选择是否需要支持跨场地？~~ → **不支持**，仅限同一场地的连续时段
- ~~列头是否显示评分/距离？~~ → **不显示**，仅场地名称 + 运动图标 + 起始价格
- ~~网格高度如何处理？~~ → **固定400px**，超出垂直滚动
- ~~如何筛选不同运动的场地？~~ → **顶部运动类型 chips**，切换后过滤显示对应场地
- ~~视频服务在哪个步骤选择？~~ → **确认前独立步骤**（Step 2），不支持视频的场地跳过
- ~~视频服务价格如何计算？~~ → **按小时叠加**，视频单价 × 预约时长 + 场地费
- ~~不同场地视频支持情况？~~ → **部分场地支持**，足球全部（直播+录像），篮球室内（仅录像），其他不支持
- ~~视频服务选择 UI 风格？~~ → **卡片式详细展示**，深色预览区 + 服务描述 + 价格
