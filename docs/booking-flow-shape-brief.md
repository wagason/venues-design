# Design Brief: 预订时间表网格视图

> Shape 命令输出 — 2026-05-08
> 最后更新 — 2026-05-08
> 目标：优化 booking-flow 步骤1的交互布局

## 1. Feature Summary

将预订流程的步骤1从"顺序选择"（场地卡片 → 日期滚动 → 时段网格）改为**时间表视图**：用户先选择运动类型，再选择日期，然后在时间表中比较该运动所有场地的可用时段。横轴为场地，纵轴为时段，每个格子显示价格和时间段。网格区域固定高度400px，超出滚动。

## 2. Primary User Action

先选择运动类型，再**同时比较该运动的多个场地在特定日期的可用时段**，快速找到合适的时间段并完成选择。

## 3. Design Direction

- **Color strategy**: Restrained — 着色中性色 + 强调色。网格本身需要大量信息承载，颜色策略保持克制，用强调色高亮选中状态。
- **Theme scene sentence**: 运动爱好者在户外阳光下，单手握手机，快速扫视多个场地的可用情况，需要在明亮环境中清晰可读。
- **Anchor references**:
  - Google Calendar 周视图（时间轴 + 多列布局）
  - 大众点评排号系统（实时状态一目了然）
  - 运动场馆小程序的时间表视图
- **Per-surface override**: 无，沿用 DESIGN.md 中的 Restrained 策略。

## 5. Layout Strategy

```
┌─────────────────────────────────────────┐
│  运动类型选择（横向滚动 chips）           │
│  ⚽足球  🏀篮球  🏸羽毛球  🎾网球  🏊游泳  │
├─────────────────────────────────────────┤
│  日期选择器（横向滚动，保留现有样式）      │
├─────────────────────────────────────────┤
│  时间表网格（固定高度400px，超出滚动）     │
│ ┌──────┬──────┬──────┬──────┐           │
│ │5人制A│5人制B│7人制A│11人制│  ← 列头    │
│ ├──────┼──────┼──────┼──────┤           │
│ │¥200  │¥200  │¥350  │¥600  │           │
│ │09~10 │09~10 │09~10 │09~10 │  ← 格子   │
│ ├──────┼──────┼──────┼──────┤           │
│ │¥200  │已满  │¥350  │¥600  │           │
│ │10~11 │10~11 │10~11 │10~11 │           │
│ └──────┴──────┴──────┴──────┘           │
├─────────────────────────────────────────┤
│  底部栏：价格汇总 + 下一步按钮            │
└─────────────────────────────────────────┘
```

- 运动类型选择器在最顶部，横向滚动 chips，默认选中足球
- 日期选择器在运动类型下方，保持现有横向滚动样式
- 网格区域**固定高度400px**，内容超出时垂直滚动
- 无独立时段列，时间信息直接显示在每个格子内（格式：`09:00~10:00`）
- 场地名称作为列头，带运动图标和起始价格
- 列头固定在顶部（sticky），滚动时保持可见
- 每个格子显示：价格 + 时间段（如 `¥200 09:00~10:00`）
- 已满格子显示"已满"
- 高峰时段：格子顶部2px暖色线条标识
- 选中状态：格子填充强调色，显示白色文字
- 支持范围选择：点击起始格子，再点击结束格子，中间格子高亮

## 6. Key States

| 状态 | 用户需要看到什么 |
|---|---|
| 默认 | 显示足球运动的所有场地时间表，无选中时段 |
| 切换运动类型 | 网格更新为该运动的场地列表，重置已选时段 |
| 切换日期 | 网格更新为该日期的可用情况，重置已选时段 |
| 选中单个格子 | 格子高亮，底部显示价格汇总 |
| 范围选择 | 起始到结束的格子都高亮，显示总时长和总价 |
| 已满格子 | 灰色背景，"已满"文字，不可点击 |
| 高峰时段 | 格子顶部2px暖色线条，价格文字用高峰色 |
| 加载中 | 骨架屏或加载指示器 |
| 无可用时段 | 空状态提示，建议切换日期或场地 |

## 7. Interaction Model

- **运动类型切换**：点击顶部 chips 切换运动类型，网格即时更新显示该运动的所有场地，切换时重置已选时段
- **日期切换**：点击日期 chips 切换日期，网格即时更新，切换时重置已选时段
- **滚动**：网格区域400px固定高度，内容超出时垂直滚动，列头 sticky 保持可见
- **点击格子**：选中单个时段
- **再次点击**：取消选中（单个）或选择范围结束点
- **范围选择**：点击起始 → 点击结束 → 自动填充中间格子 → 检查范围内是否有已满时段
- **底部栏**：实时更新价格和时段信息，"下一步"按钮根据选中状态启用/禁用

## 8. Content Requirements

- 运动类型 chips：图标 + 名称（足球、篮球、羽毛球、网球、游泳）
- 列头：场地名称 + 运动图标 + 每小时起始价格
- 格子内容：¥价格 + 时间段（如 `¥200 09:00~10:00`）或 "已满"
- 高峰标识：格子顶部2px暖色线条（不是侧边条纹）
- 范围提示："已选 3 个连续时段，14:00-17:00"
- 空状态："该日期暂无可用时段，试试其他日期或场地"

## 10. Resolved Questions

- ~~移动端屏幕较小，4个场地列是否需要固定列宽？~~ → **固定列宽**，水平滚动浏览
- ~~范围选择是否需要支持跨场地？~~ → **不支持**，仅限同一场地的连续时段
- ~~列头是否显示评分/距离？~~ → **不显示**，仅场地名称 + 运动图标 + 起始价格
- ~~是否需要独立时段列？~~ → **不需要**，时间信息直接显示在格子内（格式：`09:00~10:00`）
- ~~网格高度如何处理？~~ → **固定400px**，超出垂直滚动
- ~~如何筛选不同运动的场地？~~ → **顶部运动类型 chips**，切换后过滤显示对应场地
